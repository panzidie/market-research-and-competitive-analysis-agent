# 市场调研命令

## 输入
$ARGUMENTS

## 执行流程

### Phase 1: 需求解析
1. 解析用户的调研目标，识别：
   - 调研的行业/领域
   - 需要分析的竞品列表（明确或隐含）
   - 分析维度（功能/定价/市场/技术）
   - 报告深度和格式要求

### Phase 2: 数据采集
2. 对每个竞品，启动 `researcher` 子代理进行数据采集：
   - 搜索竞品官网、新闻报道、用户评价
   - 采集产品功能、定价、发布时间等关键信息
   - 记录所有数据来源

### Phase 3: 数据分析
3. 启动 `analyst` 子代理对采集的数据进行分析：
   - 建立功能对比矩阵
   - 进行 SWOT 分析
   - 识别市场趋势

### Phase 4: 事实核查
4. 启动 `fact_checker` 子代理对分析结果进行验证：
   - 检查数据来源
   - 交叉验证关键数据
   - 标记待确认项

### Phase 5: 报告生成
5. 启动 `writer` 子代理生成最终报告：
   - 按标准结构组织内容
   - 嵌入对比表格和图表建议
   - 标注所有数据来源

## 质量门禁
- 所有数据必须有来源 → 否则标记为"待验证"
- 关键数据必须经过交叉验证 → 否则暂停并提示人工介入
- 报告必须通过事实核查 → 否则返回修改

## 输出
生成完整的市场调研报告，保存在 `./data/reports/` 目录下。
