## 实验4（day4下午最后一个实验）：将 RAG 封装为 Skill

### 实验目标

把前面学的所有东西整合起来——用知识库检索 + 大模型生成，封装成一个可复用的 Skill 函数。

### 代码结构速览

打开 `rag_skill.py`，核心函数是第 40~83 行的 `rag_search()`：

```python
def rag_search(query: str, model=None, collection=None, top_k: int = 3) -> str:
    # ===== 第一步：向量检索 =====
    q_emb = model.encode([query])[0].tolist()
    results = collection.query(query_embeddings=[q_emb], n_results=top_k)
    docs = results['documents'][0]
    context = "\n".join(f"[{i+1}] {doc}" for i, doc in enumerate(docs))

    # ===== 第二步：组装 Prompt，调用大模型 =====
    prompt = f"""根据以下参考文档回答问题...
参考文档：
{context}
问题：{query}"""

    response = client.chat.completions.create(
        model=LLM_MODEL,
        messages=[{"role": "user", "content": prompt}],  # 把上下文塞进 Prompt
        ...
    )
    return answer + "\n\n📚 参考文档：\n" + context
```

> 这就是完整的 **RAG 流程**：检索（Retrieval）→ 增强（Augmented Prompt）→ 生成（Generation）

### 操作步骤

**步骤1：确认配置**

检查第 21 行 `LLM_API_KEY` 是否已替换。确保 `chroma_db/` 目录存在（实验1已建）。

**步骤2：运行自动测试**

```bash
python rag_skill.py
```

脚本会自动测试 4 个查询：

```
  查询: 项目完成率是多少？
  结果: [rag_search]
根据文档[1]，项目当前完成率为85%。

📚 参考文档：
[1] 议题1：项目进度。当前完成率85%，预计10月底可交付。负责人：张三。
[2] 会议决议：1) 项目进度保持，每周同步。...
[3] 财务决议：本季度预算执行率78%...
```

> ✅ **检索**：从知识库找到相关文档 → **生成**：大模型基于文档内容回答

继续观察：

```
  查询: 下个月有什么计划？
  结果: [rag_search]
根据文档[1]，下个月的计划是：启动2.0版本规划，完成需求评审，
计划10月20日前完成，负责人为王五。
```

> ✅ 答案准确、引用清晰。大模型**没有自由发挥**，而是严格基于检索到的文档回答。

```
  查询: 今天天气怎么样？
  结果: [rag_search]
我无法从已有文档中找到相关信息。
```

> ✅ 知识库没有的信息，模型**不会瞎编**，而是诚实告知无法回答。这比直接问大模型"今天天气怎么样"更可靠——RAG 让模型只基于事实回答。

**步骤3：理解 RAG = 检索 + 生成**

```
                    ┌──────────────────┐
  用户问题 ──────→  │ 向量检索知识库      │ ← 实验1/2 的核心
                    │ (找相关文档片段)    │
                    └────────┬─────────┘
                             │ 文档片段
                             ▼
                    ┌──────────────────┐
                    │ 组装 Prompt       │
                    │ "根据以下内容回答：│
                    │  [1] 完成率85%    │
                    │  问题：xxx?"      │
                    └────────┬─────────┘
                             │
                             ▼
                    ┌──────────────────┐
                    │ 大模型生成回答     │ ← 实验3 的 API 调用
                    │ (基于事实回答)     │
                    └──────────────────┘
```

### 代码对照理解

```
rag_search(query)
  │
  ├── ① 编码查询：model.encode(query) → 向量
  ├── ② 向量检索：collection.query(向量) → 相关文档
  ├── ③ 组装 Prompt：文档 + 问题
  └── ④ 调用 API：client.chat.completions.create(带上下文的 prompt)
       │
       └──→ 返回：答案 + 参考文献
```
